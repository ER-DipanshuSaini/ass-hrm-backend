# HRMS Lite Base (Backend)

A robust, production-ready REST API backend for a Human Resource Management System built with Python, FastAPI, and SQLAlchemy.

## ✨ Features

- **Employee Management**: Full CRUD operations with email and unique ID validation.
- **Attendance Tracking**: Log daily attendance with smart UPSERT validation to prevent duplicate entries.
- **Reporting Analytics**: Aggregate counts and return attendance datasets sorted chronologically.
- **Architectural Excellence**: Clean Service-Repository pattern separating `Router` -> `Controller` -> `Service`.
- **Validation**: Pydantic v2 for strict request and response schema data validations.

## 🛠️ Tech Stack

- **Framework**: FastAPI (High-performance API handling)
- **ASGI Server**: Uvicorn
- **ORM**: SQLAlchemy 2.0
- **Database**: SQLite (Zero-config for evaluation, adaptable to PostgreSQL)
- **Validation**: Pydantic v2
- **Environment Management**: `python-dotenv` & `pydantic-settings`

## 🚀 Setup & Installation

### 1. Environment Preparation
Initialize a virtual environment to keep dependencies localized:
```bash
python -m venv venv

# On Windows (Powershell)
.\venv\Scripts\activate

# On Unix/MacOS
source venv/bin/activate
```

### 2. Install Packages
```bash
pip install -r requirements.txt
```

### 3. Run the Server
FastAPI along with Uvicorn provides a lightning-fast local server.
```bash
uvicorn app.main:app --reload --port 8000
```
Server drops on `http://localhost:8000`

## 📚 API Reference (Swagger & ReDoc)

Interactive API documentation is automatically generated by FastAPI. Visit:
- **Swagger UI**: `http://localhost:8000/docs`
- **ReDoc**: `http://localhost:8000/redoc`

### Core Endpoints Overview

#### Employee APIs
- `GET /api/employees/`: Fetch all employees (ordered chronologically)
- `POST /api/employees/`: Register an employee
- `PUT /api/employees/{id}`: Update an employee record
- `DELETE /api/employees/{id}`: Delete an employee

#### Attendance APIs
- `GET /api/attendance/`: Fetch all attendance logs
- `POST /api/attendance/`: Mark/Upsert new attendance ('Present'/'Absent')
- `DELETE /api/attendance/{id}`: Remove a record

## 💡 Database Assumptions 
- Uses default SQLite local file (`hrms-lite.db`) for instant evaluations without requiring Docker/PostgreSQL installs initially.
- Pydantic models automatically validate incoming data throwing `422 Unprocessable Entity` when formats mismatch.
